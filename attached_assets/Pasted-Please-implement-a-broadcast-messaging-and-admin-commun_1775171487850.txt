2. MESSAGE A SPECIFIC USER (ADMIN → USER)
═══════════════════════════════
In the admin Users section, add a "Send Email" button next to each user.

Clicking opens a compose modal:
- To: [user's name and email] — pre-filled, not editable
- Subject: text input
- Message: textarea
- Send button

This sends a direct personal email to that specific user via Resend.

Use cases:
- Responding to a support query
- Warning about behaviour
- Account notification
- Personal welcome message

EMAIL TEMPLATE for direct admin messages:
- Dowry.Africa logo
- "Hi [Name],"
- Message body
- "Warm regards, The Dowry.Africa Team"
- Footer

═══════════════════════════════
4. BACKEND API
═══════════════════════════════
All protected by ADMIN_SECRET:

- POST /api/admin/communications/broadcast
  * Accepts: subject, body, recipientGroup, ctaLabel (optional), ctaUrl (optional)
  * Fetches recipient emails based on group selection
  * Sends via Resend in batches (max 50 per batch to avoid rate limits)
  * Logs the broadcast to broadcast_logs table
  * Returns: { sent: number, failed: number }

- GET /api/admin/communications/broadcast — returns broadcast history

- POST /api/admin/communications/user/:userId
  * Sends direct email to specific user
  * Accepts: subject, message
  * Returns success/failure

- GET /api/admin/communications/preview
  * Returns recipient count for a given group selection

6. RATE LIMITING & SAFETY
═══════════════════════════════
- Send in batches of 50 emails at a time with 1 second delay between batches
- This prevents hitting Resend rate limits
- Show progress in the UI: "Sending... 50/200 emails sent"
- If any fail, log them and report back: "195 sent, 5 failed"

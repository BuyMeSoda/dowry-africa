Please add strong password requirements throughout the app wherever a password is created or changed.

═══════════════════════════════
1. PASSWORD REQUIREMENTS
═══════════════════════════════
All passwords must meet these requirements:
- Minimum 8 characters
- At least one uppercase letter (A-Z)
- At least one lowercase letter (a-z)
- At least one number (0-9)
- At least one special character (!@#$%^&*()_+-=[]{}|;:,.<>?)

═══════════════════════════════
2. PASSWORD STRENGTH INDICATOR
═══════════════════════════════
Show a visual strength indicator below the password field on:
- Registration/signup page
- Reset password page
- Any future change password page

Strength levels:
- Weak (red bar) — less than 3 requirements met
- Fair (orange bar) — 3 requirements met
- Good (yellow bar) — 4 requirements met
- Strong (green bar) — all 5 requirements met

Show the bar as a horizontal progress bar that fills and changes color as the user types.

Below the bar show which requirements are met and which are not:
✓ At least 8 characters (green when met, gray when not)
✓ Uppercase letter (green when met, gray when not)
✓ Lowercase letter (green when met, gray when not)
✓ Number (green when met, gray when not)
✓ Special character (green when met, gray when not)

These checklist items update in real time as the user types.

═══════════════════════════════
3. VALIDATION
═══════════════════════════════
- Do not allow form submission if password does not meet all requirements
- Show clear error message if they try to submit with weak password:
  "Password does not meet requirements. Please check the requirements below."
- Backend also validates password strength — reject weak passwords even if frontend is bypassed

═══════════════════════════════
4. WHERE TO APPLY
═══════════════════════════════
- Registration/signup page — password field
- Reset password page — new password field
- Do NOT apply to login page — users should still be able to login with old passwords

═══════════════════════════════
5. DEMO ACCOUNTS EXEMPTION
═══════════════════════════════
Demo accounts use the password "demo" which does not meet requirements.
Make sure demo account login still works — the strength requirement only applies when CREATING or RESETTING a password, not when logging in.

Please implement a gated onboarding / manual approval system for new user registrations.

═══════════════════════════════
1. DATABASE
═══════════════════════════════
Add to users table:
- account_status (enum: 'pending' / 'approved' / 'rejected') default: 'pending'
- approved_at (timestamp, nullable)
- rejected_at (timestamp, nullable)
- rejection_reason (text, nullable)

Add to settings table:
- manual_approval_required (boolean, default true)

═══════════════════════════════
2. REGISTRATION FLOW
═══════════════════════════════
When a new user registers:

Check the manual_approval_required setting:

IF manual_approval_required = true:
- Create account with account_status = 'pending'
- Send email verification as normal
- After email is verified → show the "Application Under Review" page
- User cannot access Discover, Messages, Premium or any app page
- Only accessible routes: /pending, /verify-email, /logout

IF manual_approval_required = false:
- Create account with account_status = 'approved'
- Normal flow — after email verification → access the full app immediately

═══════════════════════════════
3. PENDING APPROVAL PAGE (/pending)
═══════════════════════════════
Create a beautiful full-screen page shown to pending users when they try to access any protected route.

Design:
- Dowry.Africa logo prominently at top
- Warm cream/off-white background matching the app
- Heading: "Your application is under review"
- Subtext: "Thank you for joining Dowry.Africa, [First Name]. We review every application personally to ensure our community remains intentional and serious."
- Timeline indicator showing:
  * ✅ Step 1: Account created
  * ✅ Step 2: Email verified  
  * ⏳ Step 3: Application under review (5-7 days)
  * 🔒 Step 4: Welcome to Dowry.Africa
- Message: "You will receive an email at [their email] once your application has been reviewed."
- "Questions? Contact us at hello@dowry.africa"
- Social media links: Instagram, TikTok, Twitter/X @dowry.africa
- Logout button at bottom

This page should feel premium and reassuring — not like an error page.

═══════════════════════════════
4. ROUTE PROTECTION
═══════════════════════════════
Update the frontend route protection:
- If user is logged in BUT account_status = 'pending' → redirect to /pending
- If user is logged in BUT account_status = 'rejected' → show rejection page
- If user is logged in AND account_status = 'approved' → normal access
- /pending, /verify-email, /login, /logout, /unsubscribe routes always accessible

Include account_status in the JWT token and/or GET /api/auth/me response so frontend can check it.

═══════════════════════════════
5. REJECTION PAGE
═══════════════════════════════
If account is rejected show a page:
- Heading: "Your application was not successful"
- Subtext: "Thank you for your interest in Dowry.Africa. Unfortunately your application was not approved at this time."
- If rejection_reason exists: show it
- "If you believe this is an error, please contact hello@dowry.africa"
- Logout button

═══════════════════════════════
6. ADMIN PANEL — PENDING APPROVALS
═══════════════════════════════
In admin Users section add a new tab: "Pending (X)" with live count badge.

Pending users list shows:
- Name
- Email
- Date registered
- Date email verified
- Days waiting (calculated from registration date)

Actions per user:
- "Approve" button (green) → sets account_status = 'approved', sends approval email
- "Reject" button (red) → opens modal to optionally add rejection reason, sets account_status = 'rejected', sends rejection email

Bulk actions:
- Select multiple pending users
- "Approve selected" button
- This is useful for approving batches weekly

"Registration Approval" setting:
- Toggle: "Manual approval required"

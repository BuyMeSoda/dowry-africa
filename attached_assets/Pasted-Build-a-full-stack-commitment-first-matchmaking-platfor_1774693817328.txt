Build a full-stack commitment-first matchmaking platform called Dowry.Africa with this exact structure:

MATCHING ALGORITHM (lib/matching.js):
Score two users 0-100 across 5 weighted dimensions:
1. Values alignment 30% — intent match, faith compatibility, family involvement, children preference, marriage timeline
2. Life stage readiness 25% — life stage compatibility table, serious badge boost, profile completeness
3. Cultural resonance 20% — heritage overlap, language overlap, country match, diaspora alignment
4. Practical compatibility 15% — location/city match, country match, relocation openness
5. Engagement quality 10% — recency of activity, response rate, badge status (hidden from users)

Hard filters before scoring: same user blocked, gender preference, age range, badge-only gate, intent mismatch gate.

After scoring, generate conversation prompts based on compatibility gaps (faith gap → faith prompt, low cultural → bridge prompt, high values → affirming prompt).

Feed ranking: badge holders first, then score descending, freshness boost for active in 48h, badge boost +5 points.

API ROUTES:
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/me
GET  /api/users/:id
PATCH /api/users/me/profile
GET  /api/matches/feed (free tier: 5 results, core/badge: 50)
GET  /api/matches/:id/score
POST /api/matches/like/:id (returns mutual:true if both liked)
POST /api/matches/pass/:id
GET  /api/matches/liked-me (free tier: blurred, paid: full)
GET  /api/messages
GET  /api/messages/:userId (requires core+ subscription)
POST /api/messages/:userId (requires core+ subscription)
POST /api/payments/create-checkout (Stripe or demo mode if no keys)
POST /api/payments/webhook (Stripe webhook handler)
GET  /api/payments/status

SUBSCRIPTION TIERS:
- free: 5 matches/day, no messaging, blurred likes
- core ($7/mo): unlimited matches, full messaging, see who liked you
- badge ($15/mo): everything in core + serious badge on profile + badge-only pool + ranked higher

PAYMENTS:
Use Stripe checkout sessions for subscriptions. If STRIPE_SECRET_KEY env var is not set or contains "YOUR_", run in demo mode — immediately upgrade the user without real payment and return { demo: true }.

AUTH:
JWT tokens, 30 day expiry. Demo accounts use passwordHash '$demo$' and password 'demo'. Real accounts use bcrypt hash with 12 rounds.

4. Messages — split panel, conversation list left, chat right with guided prompts

7. Checkout — Stripe payment form with card fields, order summary

FRONTEND API CLIENT (public/app.js):
- Stores JWT in localStorage as 'da_token'
- On page load: restore session, render user state into UI
- loadFeed() fetches /api/matches/feed and renders real cards
- handleLike() and handlePass() call respective API endpoints
- submitOnboarding() calls register then patch profile
- doPay() calls /api/payments/create-checkout, handles demo mode toast
- renderUserState() updates nav avatar, sidebar name/location/progress
